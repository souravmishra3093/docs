All API endpoints are authenticated using API Key and APP Id provided by Reverie Language Technologies when you sign up. Include these keys in the header of each API request as follows:

```yaml
REV-API-KEY: <YOUR-API-KEY>
REV-APP-ID: <YOUR-APP-ID>
```

### Obtaining Your API Credentials

1. **Sign Up** : Visit [RevUp](https://revup.reverieinc.com) and sign up for an account.

2. **Automatic API Key Generation** : Once you sign up, your first API key is automatically created.

3. **Access Credentials** : Your APP ID and API Key will be visible in the dashboard for use in your integrations.

### Using the API Credentials

To authenticate your requests, include the `REV-APP-ID` & `REV-API-KEY` in the headers of your HTTP requests. Hereâ€™s an example using JavaScript:

```javascript
const headers = {
  'Content-Type': 'application/json',
  'REV-API-KEY': <YOUR API-KEY>,
  'REV-APP-ID': <YOUR APP-ID>,
  'src_lang': 'en',
  'tgt_lang': 'hi',
  'domain': 'generic',
  'cnt_lang': 'en',
  'REV-APPNAME': 'transliteration',
  'REV-APPVERSION': '2.0'
};

const body = JSON.stringify({
  data: [
    "Reverie Language Technologies ltd website address is www.reverieinc.com"
  ],
  isBulk: false,
  ignoreTaggedEntities: false
});

fetch('https://revapi.reverieinc.com/', {
  method: 'POST',
  headers: headers,
  body: body
})
  .then(response => response.json())
  .then(data => {
    console.log('Response:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

```

### Best Practices for API Key Management

1. **Keep Your Key Secret**: Never expose your API key in public repositories or client-side code.

2. **Use Environment Variables**: Store your API key in environment variables rather than hardcoding it in your application.

3. **Monitor Usage**: Regularly check your API usage on the RevUp dashboard. You should be able to see the credits utilised & remaining
